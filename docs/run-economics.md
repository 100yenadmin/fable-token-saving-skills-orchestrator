# Run Economics

Match the instrument to the question.

## Mechanism Questions

Mechanism questions ask whether a cue fires, a hook blocks, a worker responds,
or a small behavior occurs. Use the cheapest faithful probe first.

Examples:

- focused unit test
- small synthetic transcript
- short worker run
- one cheap corroborating lane

Do not spend premium ruler runs on mechanism questions.

## Ruler Questions

Ruler questions ask whether a release, story quality bar, or customer-facing
claim is actually met. These need stronger evidence and better judges.

Examples:

- release readiness
- final UX taste
- story or quality scores
- customer-facing proof

Spend the expensive run where the expensive judgment is necessary.

## Record Methodology

For every scored or judged run, record:

- provider
- model or lane
- prompt or scenario
- version or SHA
- score method
- proof boundary

This is how token-saving becomes an operating system instead of a vibe.
